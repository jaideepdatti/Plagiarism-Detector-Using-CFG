Underlying Concepts

A detailed breakdown of the theoretical foundations used in this detector
1. Context-Free Grammar (CFG)
A Context-Free Grammar is a set of recursive rewriting rules (or productions) used to specify patterns of strings. It is a Chomsy Type-2 grammar. In this application, a custom grammar defines the syntax of our supported hypothetical language, covering constructs like variable assignments, loops, and conditions.

The CFG acts as the absolute source of truth — providing the structure that validates whether code segments are syntactically valid before any plagiarism comparison occurs.
2. Lexical Analysis & DFA Token Validation
Before applying the grammar, the code first undergoes lexical analysis, breaking the raw text into distinct structural tokens (such as KEYWORD, IDENTIFIER, NUMBER, OPERATOR).

We utilize a 5-state Deterministic Finite Automaton (DFA) that validates token sequences during this process. The automaton transitions through states like starting state → ID/Keyword → Operator → ID/Number, validating the token combinations to prevent illegal or malformed syntax strings from overcomplicating the parser.
3. Top-Down Recursive Descent Parsing (PDA)
The parser translates the validated token stream into an Abstract Syntax Tree (AST). We built an LL(1) Recursive Descent Parser, which is a software implementation equivalent to a Pushdown Automaton (PDA).

Operating strictly from top to bottom and left to right, this parser peeks at exactly one token ahead (LL(1)) to decide which CFG production rule to execute. It successfully reconstructs the full semantic parent-child structural representation of the original code snippet.
4. Tree Edit Distance & Longest Common Subsequence
We overcome superficial variable renaming (the most common plagiarism tactic) by comparing the ASTs directly instead of the raw text strings.

The ASTs are traversed and flattened into arrays, and we employ the Longest Common Subsequence (LCS) algorithm (similar to Zhang-Shasha approaches). This calculates the Tree Edit Distance — the minimum number of geometric operations (insertions or deletions) strictly required to transform one internal code structure into another.
5. Multi-Component Weighted Scoring
To deliver an accurate verdict that correctly identifies structural plagiarism while capping false positives, we utilize a tri-part weighted formula:
  • Structural Similarity (70%): Matches the abstract syntax structure via the calculated tree edit distance. Identifies structurally isomorphic algorithms.
  • Token Flow Similarity (20%): Matches the raw sequential logic progression of normalized tokens.
  • Identifier Equality (10%): Deducts points if variable names or constants are changed.

The structural similarity alone can only achieve 97%. The code strictly demands character-for-character equivalence across original source code to reach a 100% Exact Match verdict.
